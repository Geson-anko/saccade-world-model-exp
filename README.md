# saccade-world-model-exp

眼球運動（**saccade**）から着想を得た、画像の**世界モデル**に関する実験。

大きな（HD以上の）画像を、常に一定の小さな入力サイズで扱いながら、その画像を内部にモデル化する。エージェントは注視点と拡大率を能動的に選んで画像の一部を「見て」、観測の系列から画像全体の内部表現を組み上げる。

---

## 観測の作り方（サッカード機構）

正方形画像 `image: (size, size)` に対し、行動 `a = (p, z)` で部分を切り取る。

- `p = (x, y)`, `x, y ∈ [-1, 1]` — 中心からの注視点。`size/2` が `1` に対応。
- `z ∈ (0, 1]` — 切り取り辺の長さ。`size` が `1`、`z → 0` で 1 ピクセル。切り取り辺 ≈ `z · size`。
- 切り取った正方領域を固定サイズ `o: (s', s')`（`s' ≤ 128`）へリサイズ → これが観測 `o`。

`z = 1` は全体俯瞰、小さい `z` は局所拡大。同じ `s'` の解像度予算を「広く薄く」見るか「狭く濃く」見るかに割り振る **foveation / zoom** の機構。

**境界の扱い**
- 画像外へのはみ出しは黒パディングで許容する。
- 実装上、`image` を各辺 `size/2` だけ黒パディングして `(2·size, 2·size)` に拡張しておけば、任意の `(p, z)` の切り取りを場合分けなしに一様サンプルできる（端の特別扱いが消える）。
- 非正方 `(w, h)` の入力は、短辺を長辺に合わせて黒パディングで正方化してから扱う。

**幾何的直観** — 2 次元の画像を `(x, y, z)` の 3 次元へ投影する。`z = 1`（俯瞰）を頂点、`z → 0`（精細）を底面とする**円錐／角錐状のスケール空間**として、画像そのものをモデル化する。

---

## タスク

コンテキストとして行動・観測ペアの系列 `a_1, o_1, a_2, o_2, …` が与えられる。新たな行動 `a` に対し、対応する観測 `o` を **潜在空間で** 正確に予測できるようになることが目標。

- **汎化（in-context）** — 多数の画像で学習し、未知の画像でも「数枚のグリンプス → 未観測グリンプスの予測」を文脈内でこなす。各画像が独立した 1 エピソード（系列）。
- **自己回帰** — 履歴 `(a_{≤t}, o_{≤t})` と次の行動 `a_{t+1}` から `o_{t+1}` の潜在を予測する。順序・時間に意味を持たせる。
- **行動はランダムサンプリング** で外から与える。「次にどこを見るか」の能動ポリシーは（少なくとも当面）扱わない。

---

## 中心的な問い

> コンテキストを与えた先の系列モデルの**内部状態 `b_t`（belief）から、ソース画像そのものをデコードできるか。**

すなわち、サッカードの観測列を通じて、**画像全体を完備した内部表現として保持できるか**。評価は 2 層に分かれる。

- **層 1（局所）** — クエリ `a_{t+1}` に対する次グリンプスの潜在予測。「次に何が見えるか」を当てられるか。
- **層 2（大域）** — `t` 枚を消化した後の内部状態 `b_t` 単体から、全体画像を復元できるか。「画像全体を内部に保持できているか」。

**なぜランダムサンプリングが効くか** — 任意の位置・スケールのクエリに答える必要があるため、局所文脈だけでは次グリンプスを当てられない。結果として、モデルは大域的な画像表現を `b_t` に溜め込まざるを得ない。ランダム性が「belief を大域的に完備させる」圧力になる。

## アーキテクチャ

```
o ──► Encoder E ──► e ─┐
                       ├─► 系列モデル ──► ê_{t+1}   （潜在予測：学習対象）
a ─────────────────────┘        │
                                └─► b_t（belief = 内部状態）

[評価専用・detached]  e ──► Decoder ──► o        （グリンプス復号）
                     b_t ──► Decoder ──► image   （全体画像復号）
```

- **Encoder** `E: o → e` — 観測のみを符号化する（`a` は見ない）。
- **系列モデル** `(a_{≤t}, e_{≤t}), a_{t+1} → ê_{t+1}` — 因果的・自己回帰。
  - **主軸: minGRU 系** — 固定長の再帰状態がそのまま belief `b_t`。状態容量が `t` で増えないため、画像全体を有限次元へ圧縮する圧力が最大になり、「中心的な問い」を最も純粋にテストできる。
  - **比較: Transformer** — 容量で殴る対照。belief の読み出しには別途機構が必要（後で検討）。
- **損失** — 潜在空間の回帰（`ê_{t+1}` vs オンラインの `E(o_{t+1})`、**stop-grad なし・完全同時**）＋ 埋め込み分布への **SIGReg**（LeJEPA / LeWorldModel 系、表現の崩壊を防ぐ）。Encoder と系列モデルは同時に勾配を流して学習する。
- **デコーダ（可視化・評価専用）** — グリンプス復号 `e → o` と全体画像復号 `b_t → image` の 2 種。**いずれも detached な read-out** で、Encoder / 系列モデルへ勾配を返さない。ピクセル損失はデコーダ内部にのみ流れる。

**なぜ潜在予測か** — 生ピクセルを直接予測すると、画素レベルのズレという本質的でない誤差が表現学習を汚す。デコーダはあくまで「内部表現に画像情報が含まれているか」を測る読み出しであって、世界モデルの目的関数には関与しない。

---

## 成否の判定

1. **潜在予測誤差** — 保留グリンプスでの予測精度が学習で下がるか。
2. **可視化** — 予測埋め込み・belief をデコードし、真の crop / ソース画像と見比べる。
3. **文脈枚数 vs 予測誤差曲線** — 文脈が増えるほど誤差が下がるか（in-context 学習・belief 収束の証拠。層 1・層 2 の両方に効く）。

線形プローブ等の下流タスクは扱わない。重要なのは **内部的にモデル化できているか**。

## MVP スコープ（叩き台）

- **データ** — 第一段: FFHQ（1024²、1〜2 万枚サブサンプル。構造的で復元の成否が目視で分かりデバッグ容易）。第二段: 多様な自然画像（DIV2K + Flickr2K / Unsplash 等）で belief 機構の汎化を試す。
- **解像度** — `size = 1024`（実体はパディングで `2048²`、サンプリングは都度なので安い）、`s' = 64` から開始。`z = 1` 俯瞰 〜 `z ≈ s'/size = 1/16` で native 等倍 → 約 16 倍のズーム幅。計算コストは主に `s'`・系列長・モデル規模で決まり、ソースが大きくても効かない（＝「HD を小入力で扱う」思想どおり）。
- **系列** — 1 画像あたり `T = 16〜32` グリンプス、`(p, z)` ランダム。`t = 1` は情報ゼロのため予測対象外、予測は `t ≥ 2`。
- **モデル規模（単一 GPU 想定）** — Encoder 小型 CNN / ViT、埋め込み次元 `d ≈ 256`；minGRU 状態次元 `≈ d`；デコーダ小型。

## 用語

| 記号 | 意味 |
|---|---|
| `a = (p, z)` | 行動。注視点 `p=(x,y)` と拡大率 `z` |
| `o` | 観測（グリンプス）。固定サイズ `(s', s')` の切り取り画像 |
| `e = E(o)` | 観測の埋め込み |
| `b_t` | belief。系列モデルの内部状態 ＝ 画像の内部表現 |
| `size` / `s'` | ソース正方画像の辺長 / 観測の辺長（`s' ≤ 128`） |

## 依拠する考え方

- 潜在世界モデル — World Models (Ha & Schmidhuber)
- 表現の崩壊防止 — SIGReg / LeJEPA・LeWorldModel 系の損失設計
- 効率的な系列モデル — minGRU 系（並列化可能・固定長状態の RNN）
